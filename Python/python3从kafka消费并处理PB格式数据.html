    Python3从Kafka消费并处理PB格式数据

    missionOne 2022/12/10 Python

    # 安装依赖

    执行下面的命令安装python-kafka：

    pip3 install kafka-python

    # 消费kafka数据

    示例代码如下：

    #!/usr/bin/python3.6
# -*- coding: utf-8 -*-

from kafka import KafkaConsumer


consumer = KafkaConsumer("my_topic",
                         bootstrap_servers="172.16.100.10:9092",
                         group_id='my_consumer_group',
                         auto_offset_reset='latest')

for msg in consumer:
    print(msg)

    如果kafka中保存的是字符串或者json格式的数据，那么拿到数据后直接进行处理即可。但是因为我的数据是protobuf格式的，所以还需要下面的步骤对数据进行序列化：

    # 处理protobuf格式数据

    # 安装依赖

    执行下面的命令安装protobuf:

    yum -y install gcc gcc-c++ libstdc++-devel make build
wget https://github.com/protocolbuffers/protobuf/releases/download/v3.11.4/protobuf-all-3.11.4.tar.gz
tar zxvf protobuf-3.11.4.tar.gz
cd protobuf-3.11.4
./configure
make
make check
make install
protoc --version

    执行下面的命令安装protobuf_to_dict，这个模块将pb格式数据转换为字典数据：

    pip3 install protobuf3-to-dict

    因为我的使用的是protobuf3，所以这里安装的是对应版本的protobuf，安装这个包的目的是要用protoc命令，根据proto协议数据格式生成对应的python库文件。

    # 创建协议文件

    安装好后，我们需要创建一个协议文件，命名为*.proto，例如:

    # test.proto
syntax = "proto3";

message AgentMessage
{
    enum ServerType
	{
    	LINUX = 0;	
    	WINDOWS = 1;
    }

    int64 data_time = 1;		
    string ip = 2;		
    int32 mac = 3;			
    int32 kernel_version = 4;
}

    根据这个文件，执行下面的命令，可以生成一个库文件，后面将用这个库文件解析protobuf格式数据：

    protoc --python_out=./ ./test.proto

    生成的文件名为：test_pb2.py

    # 处理pb数据

    修改上面的消费kafka的数据代码：

    #!/usr/bin/python3.6
# -*- coding: utf-8 -*-

from kafka import KafkaConsumer
from test_pb2 import AgentMessage

from protobuf_to_dict import protobuf_to_dict

agent_msg = AgentStrMessage()


consumer = KafkaConsumer("my_topic",
                         bootstrap_servers="172.16.100.10:9092",
                         group_id='my_consumer_group',
                         auto_offset_reset='latest')

for msg in consumer:
    agent_msg.ParseFromString(msg.value)
    data = protobuf_to_dict(agent_msg)
    print(data)

    这里msg实际上是一个ConsumerRecord对象，其中的msg.value是实际消息体，bytes类型，其他的是一些如上报时间戳、offset等信息。

    这样就可以将protobuf的数据转换为字典类型的数据了。
